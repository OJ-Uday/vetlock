# ADR 0001: Working name → `vetlock`

**Status:** Accepted, 2026-07-12

## Context

The packet's working name (`depcheck-x`) collides with `depcheck` — an established npm tool
(unused-dependency checker). Publishing under that name would be search-hostile, user-confusing,
and a likely trademark/goodwill dispute. Nothing goes public before rename.

## Decision drivers

- Available on npm (`npm view <name>` returns 404).
- Pronounceable; `npx <name> diff …` reads naturally.
- No incumbent in supply-chain / security tooling.
- Short enough to badge and to type.
- Speaks to what the tool does (locks / dependencies / verification), not what it looks like.

## Candidates evaluated (2026-07-12)

Result of `npm view <name> name`:

| Name | Status | Notes |
|---|---|---|
| `depvet` | TAKEN | Existing package |
| `depsentry` | TAKEN | Also near Sentry™ trademark |
| **`vetlock`** | **AVAILABLE** | **Chosen.** Vet + lockfile — reads like what it does. |
| `capdiff` | AVAILABLE | Good but "cap" ambiguous outside our jargon |
| `tarscan` | AVAILABLE | Implies scanning, not diffing |
| `tarvet` | AVAILABLE | Weaker mnemonic |
| `updrift` | AVAILABLE | Cute; obscures purpose |
| `lockdiff` | AVAILABLE | Overloaded — sounds like text diff of lockfiles |
| `pkgdrift` | AVAILABLE | Similar concern to updrift |

## Decision

Use **`vetlock`**. Public npm package: `@oj-uday/vetlock`; installed CLI command:
`vetlock`. Repo: `github.com/OJ-Uday/vetlock`. GitHub Action: `OJ-Uday/vetlock/action`.

## Consequences

- All packet references to `<name>` / `depcheck-x` resolve to `vetlock`.
- Domain / social / trademark check is a P9 gate (before public launch), not a P0 blocker for
  private repo work.
- Config file: `.vetlock.json`. Cache dir: `~/.cache/vetlock/`. Env prefix: `VETLOCK_`.
